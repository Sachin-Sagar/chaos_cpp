Non-Linear Dynamics & Chaos Theory

This document bridges the gap between discrete mathematical maps and continuous differential equations, exploring the fundamental mechanisms of deterministic chaos.

1. The Logistic Map and Bifurcation Theory

1.1 Governing Equation

The logistic map is a discrete-time demographic model defined by the non-linear difference equation:

$$ x_{n+1} = r x_n (1 - x_n) $$

Where:

  • \( x_n \in [0, 1] \) represents the ratio of the existing population to the maximum carrying capacity of the environment at year \( n \).
  • \( r \in [0, 4] \) is a combined rate for reproduction and starvation (the driving parameter).

1.2 Fixed Points and Stability Analysis

To analyze the long-term behavior (the attractors) of the system, we seek fixed points \( x^* \) such that \( x^* = f(x^*) \). Setting \( x^* = r x^* (1 - x^*) \) yields two fixed points:

$$ x_1^* = 0 \quad \text{and} \quad x_2^* = 1 - \frac{1}{r} $$

The stability of a fixed point is determined by the derivative \( f'(x) = r - 2rx \). A fixed point is stable if \( |f'(x^*)| < 1 \).

  • For \( 1 < r < 3 \), \( x_2^* \) is stable because \( |2 - r| < 1 \). The population settles to a steady state.
  • At \( r = 3 \), \( f'(x_2^*) = -1 \). The fixed point loses stability, triggering a pitchfork bifurcation. The system now oscillates between two values (period-2).

2. The Lorenz System and Strange Attractors

2.1 Governing Equations

Unlike the discrete logistic map, the Lorenz system represents a continuous dynamical system described by three coupled, non-linear ordinary differential equations (ODEs):

$$ \frac{dx}{dt} = \sigma (y - x) $$ $$ \frac{dy}{dt} = x (\rho - z) - y $$ $$ \frac{dz}{dt} = x y - \beta z $$

Originally derived by Edward Lorenz in 1963 as a highly simplified mathematical model for atmospheric convection, the variables and parameters represent physical quantities:

  • \( x \): Rate of convective overturning.
  • \( y \), \( z \): Horizontal and vertical temperature variations.
  • \( \sigma \): Prandtl number (ratio of fluid kinematic viscosity to thermal diffusivity).
  • \( \rho \): Rayleigh number (difference in temperature between the top and bottom of the fluid).
  • \( \beta \): Geometric factor corresponding to the physical domain.

The classic chaotic behavior emerges at \( \sigma = 10 \), \( \beta = 8/3 \), and \( \rho = 28 \).

2.2 Phase Space and The Butterfly Effect

By numerically integrating the ODEs (e.g., using Runge-Kutta 4th Order) from an initial state \( (x_0, y_0, z_0) \), we trace a trajectory through 3D phase space.

What We Are Seeing:

The plot shows a Strange Attractor. The system is dissipative (the divergence of the vector field is \( -\sigma - 1 - \beta < 0 \)), meaning the volume of phase space shrinks to zero. However, the trajectory never intersects itself and never settles into a fixed point or a limit cycle. Instead, it orbits the two unstable fixed points, tracing out the famous "butterfly" shape.

This visualization directly illustrates Sensitive Dependence on Initial Conditions (The Butterfly Effect). Two trajectories starting arbitrarily close to one another will diverge exponentially over time (characterized by a positive maximal Lyapunov exponent). Because the trajectory is confined to the finite volume of the attractor but is infinitely long and non-intersecting, it inherently forms a fractal structure with a dimension of approximately 2.06.

3. The Mandelbrot Set: Stability in the Complex Plane

3.1 The Iterative Formula

The Mandelbrot set is the set of complex numbers \( c \) for which the function \( f_c(z) = z^2 + c \) does not diverge to infinity when iterated from \( z = 0 \). That is, the sequence:

$$ z_0 = 0 $$ $$ z_{n+1} = z_n^2 + c $$

Remains bounded in absolute value. If \( |z_n| \) exceeds 2, it is guaranteed that the sequence will diverge to infinity. The number of iterations required for the sequence to "escape" this radius is called the Escape Time.

3.2 The Complex Plane as a Phase Space

While the Logistic Map is 1D and the Lorenz System is 3D, the Mandelbrot set is visualized on a 2D Complex Plane. Each pixel on the screen represents a unique coordinate \( c = x + iy \).

What We Are Seeing:

The boundary of the Mandelbrot set is infinitely complex and self-similar—it is a fractal. Small changes in the value of \( c \) near the boundary lead to wildly different escape times, illustrating a form of Spatial Chaos. The "Misiurewicz points" and "bulbs" of the set are mathematically related to the bifurcation points of the logistic map through the process of renormalization.

4. Implementation and Numerical Simulation

4.1 Numerical Integration (RK4)

To simulate the continuous Lorenz system on a digital computer, we must discretize time. This project employs the 4th-Order Runge-Kutta (RK4) method, which provides a high degree of accuracy (error on the order of \( \Delta t^4 \)) compared to simpler methods like Euler integration.

The state \( \mathbf{P}_{n+1} \) is calculated as a weighted average of four incremental steps:

$$ \mathbf{k}_1 = \mathbf{f}(t_n, \mathbf{P}_n) $$ $$ \mathbf{k}_2 = \mathbf{f}(t_n + \frac{\Delta t}{2}, \mathbf{P}_n + \frac{\Delta t}{2}\mathbf{k}_1) $$ $$ \mathbf{k}_3 = \mathbf{f}(t_n + \frac{\Delta t}{2}, \mathbf{P}_n + \frac{\Delta t}{2}\mathbf{k}_2) $$ $$ \mathbf{k}_4 = \mathbf{f}(t_n + \Delta t, \mathbf{P}_n + \Delta t\mathbf{k}_3) $$ $$ \mathbf{P}_{n+1} = \mathbf{P}_n + \frac{\Delta t}{6}(\mathbf{k}_1 + 2\mathbf{k}_2 + 2\mathbf{k}_3 + \mathbf{k}_4) $$

4.2 High-Performance Rendering with SFML

Visualizing hundreds of thousands of points in real-time requires efficient handling of the graphics pipeline. Instead of issuing a draw call for every point—which would bottleneck the CPU and GPU communication—this simulation uses sf::VertexArray.

  • Batching: All points are packed into a single contiguous block of memory.
  • Single Draw Call: The entire dataset is sent to the GPU and rendered in one operation, enabling smooth zooming and panning even with 250,000+ points.
  • Coordinate Mapping: The Plot2D class dynamically translates mathematical coordinates to screen pixels, adjusting for the user's current zoom level and offset.

4.3 Performance Optimization & SIMD Laboratory

To achieve real-time interactive performance with millions of data points, this project goes beyond standard compiler flags, serving as a laboratory for SIMD (Single Instruction, Multiple Data) optimization. The implementation focuses on four key pillars of performance engineering:

A. Data Layout Reorganization (AoS to SoA)

Standard C++ often uses an Array of Structures (AoS), e.g., std::vector<Point2D>. While intuitive, this layout interleaves X and Y coordinates (XYXYXY...), which prevents the CPU from loading multiple identical components into a single register. This project implements a Structure of Arrays (SoA) layout, storing X and Y coordinates in separate contiguous blocks. This allows for Unit Stride memory access, enabling the CPU to load 4 or 8 identical mathematical components in a single clock cycle.

B. Manual AVX2 Intrinsics

While the -O3 flag enables auto-vectorization, this project utilizes manual Intel AVX2 Intrinsics (<immintrin.h>) to explicitly control the 256-bit YMM registers. By writing handwritten SIMD code, we bypass the limitations of compiler heuristics, particularly in complex mathematical loops where data dependencies might otherwise inhibit optimization.

C. Stream Vectorization (Parallel Universes)

A significant challenge in vectorizing chaotic systems like the Lorenz Attractor is loop-carried dependency: state \( P_{n+1} \) strictly depends on \( P_n \). To overcome this, we employ Stream Vectorization. Instead of calculating one trajectory faster, we calculate four independent trajectories (parallel universes) with slightly different initial conditions simultaneously in the same SIMD registers. This approach achieved a 3x speedup over the scalar implementation.

D. SIMD-Accelerated Rendering Pipeline

The graphics bottleneck often lies in the translation of mathematical coordinates (double) to screen pixels (float). The Plot2DSoA engine uses SIMD to:

  • Convert 4-8 coordinates from 64-bit double to 32-bit float precision in parallel.
  • Apply zoom and pan offsets to multiple points simultaneously.
  • Perform bounds checking against the window margins using vector masks.

This vectorized rendering approach yielded a 2x performance gain in the visualization layer.

E. Case Study: Mandelbrot Set SIMD

While the Logistic Map is too simple to see significant gains from SIMD, the Mandelbrot Set provides an ideal candidate. Each pixel requires a recursive complex multiplication ($z_{n+1} = z_n^2 + c$), which involves four multiplications and two additions per iteration. By processing 4 pixels simultaneously in a 256-bit AVX2 register, we can achieve significant throughput improvements.

A key challenge in vectorizing Mandelbrot is conditional branching: some pixels in a vector might escape the boundary (magnitude > 2) before others. We handle this using SIMD Masking, where we only increment the iteration counts and update values for lanes that are still "active."

5. Data Layouts: Array of Structures (AoS) vs. Structure of Arrays (SoA)

In high-performance computing, the way data is arranged in memory (the Data Layout) often has a more significant impact on performance than the algorithm itself. This project compares two primary layouts: AoS and SoA.

5.1 Array of Structures (AoS)

AoS is the conventional object-oriented approach. It groups all attributes of a single entity (e.g., a Point or a Complex number) into a single struct, then creates an array of those structs.

// AoS Definition
struct Complex {
    double r;
    double i;
};

// Memory Layout: [r1, i1, r2, i2, r3, i3, ...]
std::vector<Complex> numbers;

Pros: High spatial locality for operations that need both components of the same entity. It is intuitive and maintains encapsulation.

Cons: Poor for SIMD. To load four real components into a 256-bit register, the CPU must "skip" the imaginary components, leading to inefficient memory access.

5.2 Structure of Arrays (SoA)

SoA deconstructs objects. Instead of one array of structs, we have one struct containing multiple arrays—one for every component.

// SoA Definition
struct ComplexSoA {
    std::vector<double> r;
    std::vector<double> i;
};

// Memory Layout: 
// R-Array: [r1, r2, r3, r4, ...]
// I-Array: [i1, i2, i3, i4, ...]

Pros: Ideal for Parallelization/SIMD. Since identical components are contiguous, the CPU can load a block of 4 coordinates into a SIMD register with a single instruction (Unit Stride).

Cons: Accessing a single "object" requires indexing into separate memory locations, which can be less intuitive.

5.3 Performance Comparison (Compiler Optimization: O0)

To understand the intrinsic benefits of data layout without compiler intervention, we compared the implementations using the -O0 (No Optimization) flag. The results below show that even without the compiler's help, data layout matters.

Simulation AoS (O0) SoA Scalar (O0) Performance Delta
Logistic Map ~94 ms ~66 ms +30% Faster
Lorenz Attractor ~21 ms ~16 ms +23% Faster

Why is SoA faster even at O0?

At the -O0 level, the compiler generates literal translations of the C++ code with many memory accesses. The SoA layout provides better cache utilization for these simple loops because the "working set" for a single coordinate array is more compact. Furthermore, the SoA implementation in this project uses std::vector::push_back on separate arrays, which, surprisingly, can be more efficient for the memory controller than interleaving large 16-byte Point2D structures into a single vector at low optimization levels.

The SIMD Paradox at O0

Notably, manual SIMD (AVX2) intrinsics performed slower than scalar code for the Lorenz system at O0 (~37ms vs 16ms). This is because at O0, function calls to intrinsics are not inlined, and the overhead of "shuffling" data into registers manually outweighs the math savings. This highlights that SOA is a prerequisite for SIMD, but requires compiler optimization (O3) to truly shine.
